# Политика безопасности

## Поддерживаемые версии

| Версия | Поддержка |
|--------|-----------|
| Последний релиз | Да |
| Предыдущие релизы | Нет |

Обновления безопасности получает только последняя выпущенная версия.

## Сообщение об уязвимости

Если вы обнаружили уязвимость, пожалуйста, сообщите о ней ответственно.

**Email:** itprodavets@gmail.com

**Пожалуйста, укажите:**

- Описание уязвимости
- Шаги для воспроизведения
- Затронутый компонент/сервис
- Оценка потенциального воздействия

**НЕ следует:**

- Открывать публичный GitHub issue для уязвимостей безопасности
- Раскрывать уязвимость публично до её устранения

## Сроки реагирования

| Действие | SLA |
|----------|-----|
| Подтверждение получения | 48 часов |
| Первичная оценка | 7 дней |
| Выпуск исправления | Зависит от серьёзности |

## Классификация серьёзности

| Серьёзность | Описание | Реакция |
|-------------|----------|---------|
| Критическая | Возможна активная эксплуатация, риск утечки данных | Немедленное исправление, экстренный релиз |
| Высокая | Эксплуатация с умеренными усилиями | Исправление в следующем цикле релиза |
| Средняя | Требуются специфические условия для эксплуатации | Плановое исправление |
| Низкая | Минимальный риск, улучшение защиты в глубину | По возможности |

## Ответственное раскрытие

Мы следуем практике ответственного раскрытия:

1. Сообщающий связывается с нами приватно
2. Мы подтверждаем получение и оцениваем отчёт
3. Мы разрабатываем и тестируем исправление
4. Мы выпускаем исправление
5. Мы публично раскрываем уязвимость (с указанием автора отчёта, если он того желает)

## Область действия

Эта политика безопасности распространяется на все репозитории организации GZDKH:

- DKH.Platform (общие библиотеки)
- Все backend-сервисы (DKH.*Service)
- Все шлюзы (DKH.*Gateway)
- Все frontend-приложения (DKH.*.Web.UI)
- DKH.Infrastructure (Docker, скрипты, конфигурация)
